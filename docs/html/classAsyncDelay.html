Detailed Description

This class facilitates creating non-blocking delays and timeouts.

The AsyncDelay class allows you to create delays and timeouts that are non-blocking, meaning your program can perform other tasks while waiting. This improves the efficiency of the program, especially when running on a single-threaded microcontroller environment.

bool AsyncDelay::isDone ( )

Checks if the loop object's delay interval has expired.

Checks if the delay interval has been reached or exceeded.

Return values
trueif the loop object's delay interval has expired.
falseotherwise.

This function will not return true if the interval is set to zero. If a callback function is set, it will not be invoked by this method.

Unlike isReady(), this function does not automatically reset the timer. It's useful for scenarios where you have a longer block of code and you want manual control over the timer reset.

if (loop.isDone()) {
// Long-running code here...
loop.resetTime(); // Manual reset
}

If the timer is not manually reset, the function will enter an "open state", meaning it will continuously return true if interval is non-zero.

This method checks whether the delay interval has been reached or exceeded, without automatically resetting the internal timestamp. If the interval is reached and a callback function has been set, the callback function will be invoked.

This is useful for executing large blocks of code where the timer should be reset manually. If the timer is not reset manually, this function will continuously return true as long as the interval is non-zero.

if (loop.isDone()) {
// Long-running code
loop.resetTime(); // Manual timer reset
}
Note
Never returns true if the interval is zero.
Returns
true if the delay interval is reached or exceeded (and invokes the callback function if set), false otherwise.

bool AsyncDelay::isReady ( )

Checks if the loop object is ready to execute its task.

Checks if the delay interval has been reached or exceeded.

Return values
trueif the loop object's delay interval has expired.
falseotherwise.

If a callback function is set, it will be invoked automatically when this function returns true.

This function will not return true if the interval is zero.

Unlike isDone(), this function automatically resets the timer when it returns true. This is useful for quick operations.

if (loop.isReady()) {
// Quick operation here...
}

Note: This function may not be suitable for long-running code blocks, as it resets the timer automatically, reducing the effective delay before the next check.

// loop.getInterval() == 300
if (loop.isReady()) {
delay(200);
} else {
// This block will only execute for 100ms instead of 300ms.
}

This method checks whether the delay interval has been reached or exceeded, without automatically resetting the internal timestamp. If the interval is reached and a callback function has been set, the callback function will be invoked.

This is useful for executing large blocks of code where the timer should be reset manually. If the timer is not reset manually, this function will continuously return true as long as the interval is non-zero.

if (loop.isDone()) {
// Long-running code
loop.resetTime(); // Manual timer reset
}
Note
Never returns true if the interval is zero.
Returns
true if the delay interval is reached or exceeded (and invokes the callback function if set), false otherwise.
